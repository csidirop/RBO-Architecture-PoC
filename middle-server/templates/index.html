Image Access PoC

Keycloak authenticates the user, this PHP middle service applies the policy, and Apache forwards file-specific bearer tokens to the Apache file server only after this service has approved the request.

user must have the role . File token lifetime: seconds.

Demo Accounts

  • reader / reader123 can request protected files.
  • blocked / blocked123 can log in but will be denied by policy.
  • admin / admin opens the Keycloak admin console.

Simple default Keycloak users for tests. Auth will later use shibboleth.

Flow

  1. User signs in with Keycloak.
  2. Middle service checks the Keycloak role.
  3. Middle service issues a short-lived file token in a proxy cookie.
  4. Apache forwards it as a bearer header and the file server serves the image.
